# 📁 .agent - Contexto do Projeto

Esta pasta contém todos os arquivos de contexto que guiam o desenvolvimento deste projeto.
Estes documentos servem como **fonte única de verdade** para:

- **Design System** → Paleta de cores, tipografia, espaçamentos e componentes UI
- **Arquitetura** → Estrutura de pastas, padrões de código e convenções
- **Boas Práticas** → Regras de codificação, performance e acessibilidade
- **Regras de Negócio** → Funcionalidades específicas do e-commerce
- **Workflows** → Processos automatizados de desenvolvimento

## 📚 Índice de Arquivos

| Arquivo                                    | Descrição                                        |
| ------------------------------------------ | ------------------------------------------------ |
| [`design-system.md`](./design-system.md)   | Tokens de design, cores, tipografia, componentes |
| [`architecture.md`](./architecture.md)     | Estrutura de pastas, padrões e convenções        |
| [`best-practices.md`](./best-practices.md) | Regras de código, performance, acessibilidade    |
| [`business-rules.md`](./business-rules.md) | Regras específicas do e-commerce afiliado        |
| [`components.md`](./components.md)         | Catálogo e API dos componentes existentes        |

## 🎯 Como Usar

1. **Antes de criar algo novo** → Consulte `design-system.md` e `components.md`
2. **Ao estruturar código** → Siga o `architecture.md`
3. **Durante o desenvolvimento** → Aplique as `best-practices.md`
4. **Para lógica de negócio** → Verifique `business-rules.md`

---

> **Nota:** Mantenha estes documentos atualizados conforme o projeto evolui.
